# 聊天记录 — 2026-05-30

## 主题：项目启动、核心概念确立

---

### 项目方向

用户确认要做 **修仙放置游戏**，核心参考《梅尔沃放置》（Melvor Idle），设计理念为「漫长挂机、随时收益」+ 多技能并行升级 + 时间有限需决策优先级，目标用户为中国修仙题材爱好者。

---

### 已确定的设计决策

#### Melvor 要素 → 修仙映射

| Melvor 要素 | 核心机制 | 修仙化思路 |
|---|---|---|
| 非战斗技能 | 伐木/钓鱼/采矿/烹饪…独立但有资源链 | 炼丹要从采集到炼制，炼器要从矿石到法宝 |
| 战斗技能 | 攻防体+地城+Boss | 境界突破、法宝/功法、渡劫、秘境探险 |
| 水平扩张 | 所有技能平权，玩家自选路线 | 修仙者可以走丹修、器修、剑修、符修… |
| 离线收益 | 24h 离线累积 | 闭关修炼、灵石自动产出 |
| 精通系统 | 每件物品有独立精通等级 | 每种丹药/法宝/功法有熟练度 |
| 宠物 | 技能随机掉落宠物 | 灵兽/妖宠伴生 |
| 无重启 | 纯积累，不回档 | 修仙本就是不断向上的过程 |

#### 境界系统的三大作用

1. **基础属性成长** — 类似 Melvor 的「战斗等级」「生命等级」
2. **解锁门槛** — 新地图、新功能的解锁条件
3. **境界碾压** — 作为战斗中的核心压制数据

#### 单机/网游 → 双层异步社交

- **主体单机** — 个人挂机、成长，不抢资源、不抢怪，地图上不遇其他玩家 PK
- **第一层：神魂（竞争轨）** — 异步匹配 PvP，打其他玩家留下的镜像数据。快照在玩家主动突破大境界时手动确认生成。叙事包装为「已突破者留下的一缕神魂」。
- **第二层：功德（合作轨）** — 参考《死亡搁浅》的基建共享。高境界玩家留下高灵气修炼点、秘境等，后进玩家受益。前辈获得「功德」「香火」加速修炼。叙事自洽：大能须庇护后辈才能突破更高境界。
- **竞争 × 合作双轨制** — 被同一人的神魂暴打的同时，也踩在他留下的修炼点上成长，体验互相抵消
- **参与非强制** — 不影响能否修炼，只影响修炼速度，幅度不超过 10%

#### 压级机制

- 仅在大境界之间（筑基→金丹→元婴…），小境界内部不设压级
- 玩家在圆满期停留，持续打磨装备/技能/丹药，类似梦幻西游的 69/89 级卡级
- 神魂快照时机：玩家主动、精心配置后确认提交，然后才突破离开

---

### 设计协作规则

1. **一致性优先** — 新设计提案与已有决策矛盾时，助手主动指出，由用户决策
2. **杜绝双记录** — 矛盾的 A/B 方案不能同时标记为「已确定」
3. **决策即锁定** — 一旦确定，条目从待回答移到已确定，不随意更改
4. **不确定词汇捕捉** — 「可能」「可以」「有的」「也许」等模糊措辞视为未定案，标注 ⚠️ 待定，不逼用户当场决定；后续设计相关模块时主动翻出提醒
