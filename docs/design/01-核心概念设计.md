## 项目概述

- **类型**：修仙放置游戏（Idle/RPG）
- **核心参考**：《梅尔沃放置》（Melvor Idle）
- **设计理念**：「漫长挂机、随时收益」+ 多技能并行升级 + 时间有限需决策优先级
- **目标用户**：中国修仙题材爱好者

## 术语表

> 以下术语定义贯穿所有设计文档，新增内容必须遵循。`_Avoid_` 标注的词为废弃写法，不得在代码和文档中使用。

**主角**：玩家直接控制的修仙者。同一时间只能做一件事。拥有技能等级、可穿戴装备、可消耗物品。
_Avoid_: 玩家角色、Player Character

**行为**：主角或工人当前正在执行的事。有标签：战斗、修炼、采集、生产、建设。主角可执行所有标签行为，工人只能执行采集、生产、建设。每个行为引用一个技能来计算产出。
_Avoid_: 行动、任务、Activity

**技能**：被动等级数值，通过执行对应行为获得经验提升。技能等级影响关联行为的产出效率。例：行为"打坐"引用技能"吐纳术"。
_Avoid_: 熟练度、Skill Level、Proficiency

**工人**：中后期（宗门系统）解锁的自动化单位。纯数字，无技能、无装备、不消耗物品。可派遣多个工人到同一行为叠加效率。只能做采集、生产、建设类行为。
_Avoid_: 弟子、NPC、随从

**境界**：修仙主线成长阶段。分大境界（凡人、炼气、筑基…）和小境界（炼气一层~九层、圆满）。突破方式：小境界手动点击、消耗灵石等通用资源；大境界需准备特定材料、有成功率。
_Avoid_: 等级、Level

**修为**：核心成长资源，通过打坐等修炼行为获得，用于推动境界成长。溢出经验永不浪费。
_Avoid_: 经验值、EXP

**灵力**：战斗中的施法资源（MP），用于释放技能。区别于修为。
_Avoid_: 法力、蓝量、MP

**功法**：玩家构筑的核心载体。决定修炼速度、战斗倾向、资源效率。从凡人阶段即可获取基础功法，随境界解锁更高级功法。效果通过配置表定义的属性字符串匹配，公式在代码中实现。
_Avoid_: 技能书、被动、Talent

**物品** (inventory)：放在背包中的实体，以 `{ id, quantity }` 存储，部分物品有实例属性（如装备耐久）。占用背包格子。
_Avoid_: 道具、Item

**资源** (resource)：纯数值属性，无实例属性。如修为、灵石、心魔值。
_Avoid_: 货币、属性

**地图**：三层结构：大地图 → 小地图 → 子地点。每个子地点定义可执行的行为类型（采集点、战斗点等）。玩家同时只能在一个子地点活动。工人可被派遣到不同于主角的子地点。
_Avoid_: 区域、Zone

**品质**：物品的实例属性之一，影响物品的基础数值。由生产系统产出时随机决定。
_Avoid_: 稀有度、Rarity

**存档**：JSON 明文单文件，带版本号。触发时机：定时自动存档 + 退出存档。上线时离线收益直接结算后写入存档。
_Avoid_: Save、存盘

## 行为标签

所有行为归属于以下五个标签之一，每个行为引用一个技能来计算产出效率。

| 标签 | 说明 | 示例行为 |
|---|---|---|
| **战斗** | 挂机战斗，获得妖丹、灵石、功法残页 | 猎妖、秘境挑战 |
| **修炼** | 吐纳等，获得修为 | 打坐、吐纳 |
| **采集** | 采药、采矿等，获得原材料 | 采药、采矿、伐木 |
| **生产** | 炼丹、炼器等，将原材料转化为物品 | 炼丹、炼器 |
| **建设** | 洞府建筑升级、野外建筑修理 | 升级聚灵阵 |

### 行为与技能的关系

- 主角同时只能执行一个行为
- 工人只能执行采集、生产、建设类行为
- 每个行为引用一个技能来计算产出效率
- 技能等级越高，关联行为的产出越高

---

## 关系总览

- 一个**主角**同时只能执行一个**行为**
- 每个**大境界**包含多个**小境界**
- **工人**的产出 = 基础值 × 主角技能等级加成 × 建筑加成
- **功法**效果通过配置表中的属性字符串匹配，公式由代码实现
- 每个**行为**引用一个**技能**来计算产出效率
- **战斗**失败后自动切换为**打坐**

## ✅ 已确定的设计决策

### 参考模式：Melvor Idle 核心要素 → 修仙映射

| Melvor 要素 | 核心机制 | 修仙化思路 |
|---|---|---|
| 非战斗技能 | 伐木/钓鱼/采矿/烹饪…独立但有资源链 | 炼丹要从采集到炼制，炼器要从矿石到法宝 |
| 战斗技能 | 攻防体+地城+Boss | 境界突破、法宝/功法、渡劫、秘境探险 |
| 水平扩张 | 所有技能平权，玩家自选路线 | 修仙者可以走丹修、器修、剑修、符修… |
| 离线收益 | 24h 离线累积 | 闭关修炼、灵石自动产出 |
| 精通系统 | 每件物品有独立精通等级 | 每种丹药/法宝/功法有熟练度 |
| 宠物 | 技能随机掉落宠物 | 灵兽/妖宠伴生 |
| 无重启 | 纯积累，不回档 | 修仙本就是不断向上的过程 |

### 境界系统的三大作用

1. **基础属性成长** — 类似 Melvor 的「战斗等级」「生命等级」，境界提升带来生命、攻击力的增长
2. **解锁门槛** — 新地图、新功能的解锁条件
3. **境界碾压** — 作为战斗中的核心压制数据

### 离线收益参数

| 离线时长 | 效率 | 说明 |
|---|---|---|
| 0–24 小时 | 100% | 全额收益 |
| 24–96 小时 | 折扣效率 | 折率由配置表定义 |
| 96 小时以上 | 停止收益 | 不再累计 |

- 上线时离线收益直接结算后写入存档
- 修为溢出绝不浪费（突破后直接注入下一境界）
- 所有时间参数进配置表，可随时调整

### 标志的分歧点 / Out of Scope

> 以下内容已明确决定**不做**（MVP 阶段）或**永久取消**。

**永久取消**：
- 任务队列 — 采用"设后不管"模型，玩家设好行为后离开

**MVP 不做**：
- 心魔系统
- 洞府建筑系统
- 成就系统
- 工人/弟子（宗门系统）
- 画符、阵法、灵兽
- 功法深度构筑（只做 2-3 个基础功法种子验证机制）
- 法宝系统
- 多人/排行榜
- 战斗手动操作
- 地图随机/大地图移动
- 飞升/仙界/轮回
- 道侣系统
- 拍卖行
- 复杂装备词条

**通用原则**：
- 所有行为共用统一"读条"模型，每次读条完成 = 执行行为并获得结果
- 功法从凡人阶段开始存在，非筑基后解锁
- 每个阶段完成后游戏必须可运行
- 配置表优先，新增内容不应大改代码

## ❓ 待回答的关键设计抉择

> 下方每个问题都需要在后续讨论中逐一确认，防止遗漏。

### 🔴 高优先级

#### Q1：境界突破的条件是什么？
- [ ] 方案 A：独立「修炼」技能，挂机涨经验，满了就突破（最简）
- [ ] 方案 B：多技能综合门槛（功法+炼丹+炼体达到一定值）
- [ ] 方案 C：修炼经验 + 突破材料（丹药）+ 渡劫成功
- **追问**：玩家从"炼气一层到炼气九层再到筑基"，每天的操作/观察是什么样的？

#### Q2：渡劫失败是否有惩罚？
- [ ] 无惩罚（纯数值 check，类似 Melvor 风格）
- [ ] 有风险/收益紧张感（材料损失？境界倒退？冷却时间？）

#### Q3：单机还是网游？

> ✅ **已决策**：核心体验为单机，但加入双层异步社交。详见下方。

**决策内容**：

- **主体单机** — 个人挂机、成长，不抢资源、不抢怪，地图上不遇其他玩家 PK（同 Melvor）
- **双层异步社交** — 玩家之间不直接实时交互，但通过两个机制产生间接连接

**第一层：神魂（竞争轨）**

- 异步匹配战斗，PvP 场中与其他玩家留下的镜像数据对战
- 神魂快照时机：玩家手动操作突破大境界时，系统主动询问是否留下一缕神魂；玩家精心调整配置后确认提交，快照生成，然后离开此境界
- 叙事包装：已突破的玩家在此境界留下的一缕「神魂」战斗意志

**第二层：功德（合作轨）** ⭐ 核心差异化设计

- 参考《死亡搁浅》的基建共享：老玩家可以在世界中留下有益的东西
- 修仙化后：前世大能留下的**高灵气修炼点**、**秘境**等
- **后进玩家受益** — 不会觉得「玩晚了永远落后」，反而感激前辈的遗留
- **前辈玩家收益** — 用修仙概念「功德」「香火」包装；高境界大能须对低境界修士挥洒恩泽、积累功德，才能更快修炼
- **叙事自洽** — 大能不是自闭挂机，而是需要通过庇护后辈来突破更高境界

**竞争 × 合作双轨制**：被同一人的神魂暴打的同时，你也踩在他留下的修炼点上成长——两股体验互相抵消，避免「仇老」情绪

**参与非强制**：神魂和功德均为可选系统，不参与不影响能否修炼，只影响修炼速度，影响幅度不超过 10%

**压级机制** — 玩家可在某大境界「圆满期」停留，持续打磨装备/技能/丹药，类似梦幻西游的 69/89 级卡级

> ⚠️ 待定：PvP 具体奖励内容（灵石/荣誉/天梯？）、功德/香火的具体数值机制、秘境/修炼点的生成规则

### 追加设计 1：Bazaar 式技能法宝战斗 ⭐ 核心差异化

**对比 Melvor**：Melvor 战斗 = 数值碾压 + 攻防石头剪刀布，策略浅

**本作设计**：

- 功法/法宝**永久收集**，数量可达几十上百
- 出阵有 **X 个法宝位 + Y 个功法位**（有限 slots），战前自选配置
- 每场 Boss 战是一个**解谜**——用有限 slots 搭出应对方案
- 死亡不是结束，是信息：回去看日志，判断是防御不够还是攻击不够，调整 build 再挑战

**参考 The Bazaar 的核心战斗规则**：

| 机制 | 规则 | 修仙映射 |
|---|---|---|
| 冷却制 | 每个道具独立冷却秒数，冷却完毕自动触发 | 功法/法宝的灵力蓄力 |
| 加速 | 冷却速度 ×2 | 加速类功法/轻功 |
| 减速 | 敌方冷却速度 ÷2 | 迟缓/困敌法术 |
| 冻结 | 敌方道具完全停转 | 封禁/定身术 |
| 充能 | 一次性缩短冷却（爆发式） | 灵气爆发 |
| 多重施法 | 一次触发打多次 | 剑诀连击/分光化影 |
| 灼烧 | 每秒跳 2 次伤害，逐减 | 火系功法 |
| 中毒 | 每秒伤害，无视护盾 | 毒系功法 |
| 护盾 | 挡伤害，减半灼烧 | 护体真气 |
| 治疗/再生/吸血 | 多样恢复手段 | 回春术/血魔功 |
| 棋盘站位 | 相邻/最左/最右影响效果 | 阵法、功法搭配位 |
| 道具尺寸 | 小/中/大，占不同格数 | 法宝/功法占用神识格 |

**五行系统融入**：在以上基础上叠加五行相生相克，提供更深层的克制策略

**与放置游戏的契合**：战斗自动结算（同 The Bazaar），玩家只需要做战前配置——完美兼容放置循环

> ⚠️ 待定：法宝位/功法位的具体数量、五行克制系数、战斗日志的详细程度

### 追加设计 2：自建宗门（对标 Melvor Township，但全面升级）

**对比 Melvor Township**：经查证，Township 是 Melvor 最被诟病的技能——被玩家形容为「无聊」「新手陷阱」「不如删了」「就算不存在什么都不会改变」

**本作设计**：

- 玩家从孤身一人 → 建立宗门 → 拥有一群弟子
- **小弟分配策略**：有限弟子分配到不同资源渠道（采矿、采药、狩猎等），最大化资源产出
- **供需匹配**：宗门产出需要和自身日常消耗匹配，避免某资源过剩或短缺
- **核心策略博弈**：资源投入自身修炼（即时反馈）vs 投资宗门建设（远期回报）

| 策略维度 | 投自身 | 投宗门 |
|---|---|---|
| 反馈速度 | 即刻战力提升 | 延迟但持续放大 |
| 风险 | 0 风险，确定性收益 | 投资回收期长 |
| 天花板 | 受限于个人时间 | 人越多产出越大 |
| 叙事感 | 独行侠 | 开宗立派 |

> 💡 设计的优越性：这个二选一张力天然制造了玩家之间的路线分化——有人速冲境界（独狼），有人广积粮缓称王（宗主）。这为异步 PvP 提供了更多样的神魂快照。

> ⚠️ 待定：弟子数量上限、弟子获取方式、宗门建设的具体资源品类、投资回报曲线的数值

### 世界观框架 ⭐

> ✅ **框架已确定**（2026-06-07）。详细内容待展开为独立文档。

**核心设定**：这是一个「灵气真实存在」的 if 线世界——科学不存在，神秘能量是世界的底层物理。

- **统一能量体系**：所有文明的修炼本质都是操控同一种神秘能量，只是叫法不同（神州称灵气、北欧称符文之力、希腊称神性）——同一套底层规则的不同流派
- **架空仿古时代**：类似《九州》的虚构历史时序，非真实朝代。但使用真实地名（神州：长安、泰山、黄河等），赋予熟悉感与神秘性
- **全球版图**：地图包含全世界，各文明有其对应的神话传说和修炼体系。目前仅设计神州，海外文明后续扩展
- **出生点**：架构上支持选择海外文明作为出生地，但初期仅开放神州
- **神话层**：各地古代神话传说赋予现实地点神秘性（收集各地传说故事来构建）

> ⏳ 待展开：神州境内地点与神话的对应、灵气分布规则、各文明修炼体系的具体差异、世界观独立文档

### 🔷 七大核心乐趣（为什么玩家挂几个月/几年不腻）

#### 1. 深度技能互联网

20+ 技能互相依赖，形成复杂的资源链：

- 采矿 → 锻造 → 近战装备
- 伐木 → 制弓 → 远程
- 钓鱼 → 烹饪 → 战斗食物
- 种植 → 草药学 → 战斗药水
- 星象学 → 被动 buff → 所有技能

→ 这创造了一个开放性优化问题：**永远有更高效的路径等着你发现**。玩家不是在执行任务清单，而是在不断解决「我现在最优做什么」的自主决策。

#### 2. 自主目标设定

游戏没有强制的 A→B→C 主线。正如一位玩家所说：「这个游戏关注的是旅程中的成就，而不是终点本身。」

→ 心理效果：**永续的不完整感**。你永远感觉「还有一个最优选项可以去做」。这和《异星工厂》《星露谷》的驱动力同源——游戏永远不会对你说「你通关了」。

#### 3. 多层并行成长轨

| 轨道 | 说明 |
|---|---|
| 技能等级 | 1→99（DLC 到 120） |
| 精通系统 | 每件物品独立精通 1→99 + 精通池 + 检查点 |
| 装备收集 | 稀有掉落、Boss 掉落、地城奖励 |
| 宠物收集 | 技能训练中随机掉落的 40+ 宠物 |
| 地城完成 | 逐步解锁更深层地城 |

→ 多轨意味着「停不下来」——你刚达成技能 99，精通池还没满；精通满了，宠物还没出。

#### 4. 精通池的张力设计 ⭐ 关键机制

- 精通 XP 的 25%（技能满级后 50%）进入共享池
- 可以用池子 XP 直接升级任意物品
- 但池子有检查点（10%/25%/50%/95%），达到阈值给被动 buff
- 花掉池子 XP，buff 可能消失

→ **把被动系统变成了主动资源管理**：花 or 不花？永远在博弈。

#### 5. 离线收益 = 信任关系

非战斗技能 24h 离线累积。你回来时是「发现收获」而不是「被惩罚没在线」。

→ 建立了一种少见的玩家-游戏关系：不是 FOMO（错失恐惧），而是 **期待回归**。

#### 6. 变率奖励层

稀有掉落（掠夺护符、神剑、宠物）不期而至，叠加在稳定成长之上。

→ 双轨多巴胺：稳步成长的满足 + 随机掉落的惊喜。

#### 7. 「心眼」效应

Jagex 的 Chris Pfeiffer 评价：

> 「你在脑海之眼中构建了一幅关于你角色正在做什么的画面。而且它非常个人化。」

→ 极简 UI 不是弱点，而是**想象力的画布**。数字撑起叙事。

## 🔪 投资人灵魂拷问（待回答）

> 以下六个问题来自投资人/毒舌玩家视角的严厉反问。每个都指向设计的潜在内在矛盾，需逐一想清楚。

### Q-invest-1：战斗策略 × 放置是矛盾的？

> ✅ **已回答**（2026-05-31）

Bazaar 式战斗需要「试 build → 看回放 → 调整 → 再试」，Boss 战可能反复调整。放置游戏核心是「设好 → 挂机 8h → 收菜」。每 2 小时当一次战术策划，这叫放置游戏吗？

**答案**：参考《剑与远征》（AFK Arena）的挂机+战前策略模型。

- **挂机决定上限（80%）** — 战力主要来自挂机积累（等级、装备、境界），过不去就睡一觉
- **策略决定效率（20%）** — 战前功法法宝 BD、五行克制帮你打比你高 ~20% 战力的关，但不是必须的
- **不强制策略** — 完全可以把策略层当不存在，纯挂机碾压过去，只是慢一点
- **核心原则**：策略是加速器，不是门槛。Boss 打不过？两条路：重调 BD 精妙破局（策略路径），或挂几天升境界碾压（放置路径）。区别只是速度和玩家偏好。

> 🔒 **框架约束**：挂机 80% / 策略 20%，后续设计战斗时若策略占比过大，助手需提醒。

### Q-invest-2：压级 + PvP = 为什么我要突破？

> ✅ **已回答**（2026-05-31）

如果在筑基圆满打磨到极致，筑基 PvP 无敌，突破到金丹反而从头挨打。最理性选择是永远不突破——功德照拿，宗门照滚。如果压级能称霸 + 突破才解锁新内容同时成立，PvP 匹配池全是万年压级卷王，新手怎么玩？

**答案**：参考梦幻西游 69→89 的突破动力分析。在此游戏中，梦幻西游的三大突破阻力中有两个不存在：

- **无成本阻力**：挂机游戏不氪金，法宝功法永久收集，突破不需要换装备
- **无阶级跌落**：前境界已压满，神魂已强，持续提供异步 PvP 收益
- **逆转逻辑**：不升境界 = 永远挂低级图 = 资源效率越来越低。突破解锁高级图，挂机收益更高
- **异步匹配**天然不依赖在线人数，不存在「高段位找不到人」

**突破的动力**：解锁新功法/法宝/秘境/功能 + 高级图资源效率更高 + 神魂持续产生被动收益 + 新境界 PvP 池全新竞争环境

> ⚠️ 待定：玩家突破离开某境界后，是否还能回去编辑该境界的功法法宝 BD？这影响神魂的后续可优化性。

### Q-invest-3：宗门投资博弈可能是伪命题

> ✅ **已回答**（2026-05-31）

「投自身 vs 投宗门」——在一个时间 × 产出累积型游戏里，复利碾压一切。如果宗门前 3 天效率 +20%，第 4 天起这个 20% 差额永远滚下去。第 30 天的宗门回报已碾压自身修炼。这不是有趣的策略选择，是小学数学题，答案永远是倾斜给复利最快的一方。

**答案**：宗门产出**独占物品**——功法/法宝（核心战斗能力）。不去和复利数学硬刚，而是换赛道。

- 功法/法宝 = 核心战力 → 必须获取 → 只能从宗门获取 → 无论 ROI 多低都得投
- 从「比谁复利快」的数学题，变成「必须投，投多少、何时投」的策略题
- **分类澄清**：宗门是核心养成系统（单人、强制），不是社会系统。社会系统（神魂/功德）的非强制 10% 规则不适用于宗门。

### Q-invest-4：你没有 RuneScape 情怀获客引擎

> ✅ **已回答**（2026-05-31）

Melvor 有 20 年 RuneScape 玩家基础——技能名、装备名、世界观现成的。你原创修仙 IP，怎么让修仙读者觉得「这就是我要的感觉」，又让 Melvor 玩家觉得「这值得我放下 Melvor 来玩你」？两边讨好可能两边不讨好。

**答案**：主要目标用户是**修仙小说读者**，不是 Melvor 用户。

- 修仙概念必须来自最广泛传播作品的**最大公约数**（如《凡人修仙传》《仙逆》《一念永恒》）
- 70% 概念让读者觉得熟悉，30% 来自其他作品但不违背认知
- 目标是「味道最对」而非「两边讨好」
- 详细的世界观概念调研、公约数提取留到【世界观】话题时展开

### Q-invest-5：内容产能——你到底做不做得动？

> ✅ **已回答**（2026-05-31）

Melvor 技能物品种类模板化（铜剑→铁剑→钢剑…）。你的 Bazaar 式战斗里每个功法/法宝要独特冷却、效果类型、触发条件、五行属性、combo 联动。一百个功法的设计量远超一百把剑，再加宗门系统、功德系统。打算做多少内容？一个团队还是你自己？

**答案**：采用组合生成方法论（来自《背包闯江湖》经验）。

- 先设计几套**典型输出节奏模板**（开场爆发、攒 buff 爆发、拖后期等）
- 分配到不同**五行**上
- 投放到不同**部位/槽位**
- 节奏模板 × 五行 × 部位 = 大量差异化工法，设计成本可控且产出不模板化

### Q-invest-6：冥想 vs 紧张——乐趣模型互相削弱？

> ✅ **已回答**（2026-05-31）

Melvor 玩家刷数字进入冥想式放松——平静、重复、满足。Bazaar 战斗是紧张的解谜——试错、分析、策略优化。「修仙的宁静致远」和「这个五行 combo 冷却差 0.3 秒调不对啊操」能共存在同一个产品里吗？

**答案**：两种体验由玩家的**真实生活节奏**自然切分，不共存于同一时刻。

- **白天**（工作/上班）→ 挂机收菜 + 轻度操作（换挂机点、宗门指令）→ 冥想模式
- **晚上**（空余时间）→ PK + 调 BD + 反复挑战 Boss → 紧张模式
- **自然闭环**：打不过 → 睡一觉挂机碾压 → 明天再来 → 不产生持续挫败感

## 设计原则

### 不要做成频繁点击游戏

这是放置游戏，不是收菜游戏。玩家应该可以：

> 设置目标 → 离开 → 回来查看结果 → 调整下一步目标

而不是：

> 每 3 分钟点一次 / 每 5 分钟收一次 / 每 10 分钟切换一次任务

### 不要让玩家必须查攻略

游戏内应该显示：
- 当前目标
- 缺少材料
- 推荐获取地点
- 战斗胜率
- 突破成功率
- 失败风险
- 当前行动预计收益

### 不要一开始堆太多系统

修仙题材很容易无限扩张，但 MVP 只需要证明：

> 境界主线成立 → 技能资源循环成立 → 离线收益成立 → 突破目标成立

其他系统应在 MVP 之后逐步加入。

### 玩家回来时要有清晰反馈

每次重新进入游戏，都应该告诉玩家：
- 离线了多久
- 获得了什么
- 技能提升了什么
- 当前目标推进了多少
- 下一步建议做什么

这是放置游戏的关键体验。

## 📢 推广/内容策略：解决「静态界面难传播」问题

> ⚠️ 待定：方案已列出，细节回头细化

**背景**：放置游戏画面通常是静态进度条+文字，难以制作吸引人的推广内容。经查，Melvor 几乎无内容创作者生态（YouTube/Bilibili/Twitch 搜索结果均近乎为空），靠 RuneScape 情怀和口碑传播。

**五项方案**：

1. **突破/渡劫微演出** — 天劫落下、雷光劈身、护体真气明灭的几秒过场动画+音效+文字。不需要精美画面，确保观众理解「这人在渡劫」。UP 主可剪成高光片段。
2. **神魂 PvP 战斗回放** — Bazaar 式战斗自动但可回放。冷却条运转、五行动画闪烁、combo 触发特效——截 30 秒最激烈的部分即可分享：「筑基圆满最强功法配置，没打过这个神魂」。
3. **叙事取代画面** — 修仙文读者本身不需要画面。UI 用文字+少量图标讲好突破、战斗、秘境探索的故事。类似《太吾绘卷》出圈路线：故事性本身就是内容。
4. **宗门/BD 截图分享** — 《足球经理》玩家爱秀阵容，《环世界》玩家秀基地。宗门弟子分布图、功法法宝搭配面板——有识别度的截图就是社交货币。
5. **⭐ AI 修仙自传** — 玩家游戏历程（突破时机、渡劫结果、PvP 胜负、宗门发展、关键法宝）→ AI 生成章回体修仙小说 + CG 插图。玩家分享「我的修仙自传」链接，读者看到小说+插画而非进度条。每个玩家故事天然不同。UP 主可直接念小说做成视频内容。

> 核心理念：推广靠**可分享的瞬间**而非可连载的内容。五项方案覆盖截图/短视频/AI 小说三种分享形态。
